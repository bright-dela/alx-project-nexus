‚ö° Intelligent Caching
Optimized for performance and scalability

Three-Tier Cache Strategy

Nexus implements a sophisticated caching system with dedicated Redis databases for different concerns, ensuring optimal performance and preventing cache pollution.

üõçÔ∏è Product Cache (Redis DB 2)
Stores category trees, brand lists, product listings, and product details. Automatic invalidation via Celery when products or categories change. Smart key generation based on query parameters for accurate cache hits.
üîê Authentication Cache (Redis DB 3)
Manages OTP codes with ten-minute TTL, failed login attempt counters, account lockout status, and JWT token blacklist with automatic expiration.
üìã Default Cache (Redis DB 1)
General-purpose caching for session data and other application needs, keeping authentication and product caches isolated and performant.
